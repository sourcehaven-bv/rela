title: 'Authorization model: four-layer with delegate-X, union semantics, write-time-only Lua'
context: 'rela-server is increasingly used in multi-user contexts. With Principal{User, Tool} stamped at every write (TKT-WEBI, PR #767), the next question is what to do with that identity. The data-entry server today is principal-aware but uniformly privileged. A cross-system research sweep (Plone, Casbin, OpenFGA/Zanzibar, Cerbos, Oso, Postgres RLS, AWS IAM, Django Guardian, Apache Ranger, Neo4j) produced consistent signals on what works at our scale (2–20 principals × 100–10k entities) and what fails.'
consequences: 'Adopts a four-layer model — users, groups, roles, local roles — inspired by Zope/Plone and validated by OpenFGA/Zanzibar''s relationship-based modeling. Specific choices: (1) union + explicit-deny semantics (most-permissive grants win); IAM/Azure/K8s/Plone converge here, specificity ordering produces undebuggable denials. (2) Plone''s delegate-X permission pattern for tamper resistance — granting role X requires holding delegate-X; no separate ACL store or hardcoded admin. (3) Lua escape hatch only at write-time via the existing automation engine; never on the read path. Casbin/Postgres-RLS/Neo4j all demonstrate that per-row Lua is the perf cliff; we don''t take that footgun. (4) Static (acl.yaml: roles, assignments) vs dynamic (graph: role-conferring relations) split. Casbin proves static needs to stay small (~100 rules); OpenFGA proves dynamic should be graph edges. (5) Containment inheritance opt-in per relation, depth-capped at 5 (Plone''s unbounded acquisition is the warning). (6) Read filtering separates entity visibility from property-level redaction (Apache Ranger''s row-filter vs column-mask). (7) Every 403 names the rule that fired; every list response carries filtered_count. AWS IAM and Postgres RLS both fail by being opaque about denials and counts. (8) MCP integration is transport-layer intersection: effective = intersect(user_capabilities, agent_scope), default-deny on writes. Staged: v0 declarative write-side, v1 read filtering + groups + MCP intersection, v2 containment inheritance + per-property writes + explicit-deny, v3 documented automation patterns (no new Lua). Full design at .ignored/acl-design.md.'
date: "2026-05-19"
status: accepted
