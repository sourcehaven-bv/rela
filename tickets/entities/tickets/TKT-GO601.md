## Problem\n\n`rename.Rename` is the most-orchestrated writer in the codebase. It hand-rolls validate → `repo.Transaction` → mutate-graph-in-memory across ~280 lines. The new `workspace.WithTx` primitive (TKT-PNPI, PR #351) was designed exactly for this pattern.\n\nMigrating rename to use `WithTx` proves the primitive works end-to-end on a non-trivial caller AND collapses ~150 lines of orchestration into ~50 lines of intent.\n\n## The import-cycle wrinkle\n\n`workspace` imports `rename` (to expose `Workspace.RenameEntity`), so `rename` cannot import `workspace`. The current `rename` package defines a `Store` interface that `*workspace.Workspace` satisfies, avoiding the cycle.\n\nThe cleanest fix is to **move the orchestration into the workspace package** and reduce `rename` to a types-only package (Result, RelationRef, Options). The data types stay where consumers expect them; the behavior moves to where it belongs.\n\n## Scope\n\n**In scope:**\n\n1. Move the rename orchestration from `internal/rename/rename.go` into a new file `internal/workspace/rename.go` as a method on `*workspace.Workspace`.\n2. Reduce `internal/rename/rename.go` to type definitions only: `Options`, `Result`, `RelationRef`. Delete `Store`, `Rename`, `executeRename`, `updateGraph`, `writeUpdatedRelations`, `validate`.\n3. The new `Workspace.Rename(entityType, oldID, newID, opts)` uses `ws.WithTx` for the persist + graph-update phase. The validate + dry-run-result-build phase happens before WithTx (read-only).\n4. Update the existing `Workspace.RenameEntity` wrapper to call the new method.\n5. Move the existing rename tests from `internal/rename/rename_test.go` into `internal/workspace/rename_test.go`. Keep them as a regression suite — same test bodies, same assertions.\n6. Keep the public types (`rename.Result`, `rename.Options`, `rename.RelationRef`) so external callers (CLI rename command, tests) keep working.\n\n**Out of scope:**\n\n- Migrating CreateEntity/UpdateEntity/etc. to WithTx — separate ticket, requires behavior-change review.\n- Migrating importer to WithTx — separate ticket.\n- TKT-CXHM (automation re-entrance) — not relevant to rename, which doesn't trigger automation.\n\n## Acceptance Criteria\n\n1. The orchestration functions (`Rename`, `executeRename`, `updateGraph`, `writeUpdatedRelations`, `validate`) are deleted from `internal/rename/rename.go`.\n2. New `Workspace.Rename` method uses `ws.WithTx` and is materially shorter than the original.\n3. `internal/rename/rename.go` contains only type definitions (Result, Options, RelationRef).\n4. All existing rename tests pass against the new code path (relocated into the workspace package).\n5. `Workspace.RenameEntity` (the existing wrapper) still has the same external signature.\n6. CLI `rela rename` command still works end-to-end.\n7. `go test -race ./...`, `just lint`, `just coverage-check` all pass.\n\n## Risk\n\n- **Test relocation may surface hidden test dependencies** on the rename package's internal structure. Plan: copy the tests verbatim first, fix any imports, then verify.\n- **`Workspace.RenameEntity` returns `*rename.Result`**. Keep that type in the rename package as a DTO; the workspace produces it. No external API change.\n- **Effort: s.** Maybe 2-3 hours: ~250 lines of code moved + tests relocated + verify.
