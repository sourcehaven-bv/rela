id: RR-1F26
type: review-response
title: Logging policy for ErrEncrypted unspecified — risk of log-spam
finding: 'fsstore detection produces ErrEncrypted on every read of an encrypted file. With SSE refresh / file-watcher polling, this becomes one log line per file per refresh — log spam in any project with encrypted files. Spec: log once per (path, process-lifetime) using a sync.Map-backed dedupe set, or log at debug-level only. Trivial but bites in production.'
severity: nit
resolution: 'Logging policy: detection produces no per-read log line (silent path — the entity loads with Inaccessible populated, that IS the signal). Validator emits a debug-level log when skipping a rule due to inaccessible field; debug-level prevents production spam. No sync.Map dedupe needed.'
status: addressed
