title: r.Host spliced into app CSP without validation — defense-in-depth (not browser-exploitable)
finding: 'DOWNGRADED from critical after checking deliverability. appBaseURL (apps_handler.go:49) splices raw r.Host into the app''s CSP base, and Go''s server does accept Host headers containing '','' "''" ''*'' '';'' over the wire (verified). BUT: a browser will never EMIT those chars in a Host header — they can''t appear in a real DNS hostname, and the browser derives Host from the resolved origin. DNS rebinding (the only way to point a victim''s browser at a 0.0.0.0-bound rela with the Host gate off) still yields a VALID domain name (letters/digits/hyphen/dot) — none of the CSP-significant chars. The only way Host:''a,b'' reaches the server is a non-browser client (curl/raw socket) = the attacker''s OWN request served to the attacker''s OWN context; no victim, and rela has no session to ride, so no privilege is gained. Residual: a rebinding attacker can set Host to a valid-but-wrong name, making script-src point at the wrong origin — that BREAKS the app''s own scripts (not an escalation), and is the same DNS-rebinding exposure the whole server already has in 0.0.0.0-no-allowlist mode (documented trusted-LAN-only). NET: minor hardening, not a security hole. FIX (cheap, still worth doing): validate r.Host (reject chars outside [a-zA-Z0-9.:\[\]-] → 400) before it enters the CSP, so the ''CSP is the boundary'' claim is airtight regardless. Add a test. Thanks to the user for pushing on deliverability.'
severity: minor
resolution: 'appBaseURL now validates r.Host (hostUnsafeForCSP regex rejects anything outside [a-zA-Z0-9.:[]-]) and returns ok=false; handleV1App responds 400 ''bad_host'' rather than splicing an unsafe value into the CSP. So the ''CSP is the boundary'' property is now airtight by construction regardless of the (non-browser-reachable) Host. Test: TestAppBaseURL_RejectsCSPUnsafeHost covers a,b / a''b / a*b / a;b / space / empty; TestAppBaseURL confirms valid hosts incl. bracketed IPv6 still work.'
status: addressed
