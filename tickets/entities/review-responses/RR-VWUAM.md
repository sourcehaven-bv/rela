# Finding

`titleSlug` only keeps ASCII `[a-z0-9]`:

```go
case r >= 'a' && r <= 'z', r >= '0' && r <= '9':
```

Pandoc, goldmark's `auto-heading-id`, and markdown-it-anchor all preserve
Unicode word characters. Resulting anchor mismatches:

| title | this code | Pandoc/goldmark |
| ----- | --------- | --------------- |
| `Émile` | `mile` | `émile` |
| `café résumé` | `caf-r-sum` | `café-résumé` |
| `東京` | (empty) | `東京` |

Anchors generated by `entity_refs` will not match heading IDs in documents whose
headings come from non-ASCII titles → dead links.

The docs (`docs/lua-scripting.md:736-739`) explicitly claim Pandoc / goldmark
compatibility. That's currently false.

# Resolution

Use `unicode.IsLetter` and `unicode.IsDigit` instead of the ASCII class. Also
need Unicode-aware lowercasing — `strings.ToLower` already handles that.

Reference: see goldmark's `auto-heading-id.go` for the canonical algorithm.
Pandoc's algorithm (https://pandoc.org/MANUAL.html#extension-auto_identifiers):

1. Remove all formatting, links, etc.
2. Remove all footnotes.
3. Remove all non-alphanumeric characters except underscores, hyphens, and periods.
4. Replace all spaces and newlines with hyphens.
5. Convert all alphabetic characters to lowercase.
6. Remove everything up to the first letter (anchors must start with a letter).

Note step 6: anchors must start with a letter — current code drops the
empty-string case but doesn't enforce starting-letter. For pure-numeric titles
like `2024-01-15`, Pandoc emits `section-2024-01-15` whereas this code would
emit `2024-01-15`. Decide whether to match Pandoc here or document the
divergence.

Add Unicode title tests.
