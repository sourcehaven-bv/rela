title: BackButton insertion causes header layout jolt without a left-group wrapper
finding: 'EntityList.vue:458, KanbanView.vue:316, SearchView.vue:462, AnalyzeView.vue:124 insert <BackButton v-if="backTarget"> as first child of a flex header with justify-content: space-between. 2 children (h1+action) vs 3 children (BackButton+h1+action) produces different positioning — h1 snaps to center when BackButton appears. AnalyzeView''s align-items: flex-start makes BackButton sit at top of multi-line h1+subtitle. DocumentView.vue:117-119 already wraps its BackButton in .header-left; copy that pattern across views. E2E only asserts visibility, not position.'
severity: minor
resolution: Wrapped <BackButton> + <h1> in <div class="header-left"> across EntityList.vue, KanbanView.vue, and AnalyzeView.vue. Added .header-left { display:flex; align-items:center; gap:12px; } (align-items:flex-start for AnalyzeView where the heading has a multi-line subtitle). h1 now stays pinned left regardless of BackButton presence; no layout jolt. SearchView.vue uses a gap-spaced flex header without space-between, so the wrapper wasn't needed there — documented inline.
status: addressed
