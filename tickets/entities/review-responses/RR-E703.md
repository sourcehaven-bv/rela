type: review-response
title: withReadGate not attached when upstream pre-stamps acl.WithRequest
finding: 'attachACLRequest''s short-circuit `if acl.FromContext(ctx) != nil { next.ServeHTTP(...); return }` exits BEFORE withReadGate. A test (or future middleware) that pre-attaches a Request via acl.WithRequest but does NOT also attach a readGate leaves the handler with nopReadGate from readGateFromContext — every read becomes AllowAll for those callers. The new gateCtxFor helper does both, but the rest of the codebase has plenty of acl.WithRequest-only call sites. Fix in middleware: when an existing Request is found, also call withReadGate(ctx, aclReadGate{req: existing}) before forwarding. Makes ''Request attached upstream'' and ''Request attached by middleware'' indistinguishable to downstream handlers.'
severity: significant
resolution: 'attachACLRequest short-circuit branch now also calls withReadGate(ctx, aclReadGate{req: existing}) before forwarding. Upstream-Request-only callers no longer silently see nopReadGate.'
status: addressed
