title: Bridge verb allow-list omits relations, schema/config, templates, documents, dry_run, position; includes a non-existent trace
finding: 'The planned closed RPC allow-list (list/get/search/trace/analyze/create/update/delete/rename/action) is both too narrow and partly wrong: (1) RELATIONS CRUD is entirely missing — relations are a separate sub-resource (POST/PATCH/DELETE /{plural}/{id}/relations/{relType}[/{targetId}], entities.ts:197-251) handled by distinct backend handlers (api_v1.go:317-340), and the backend''s own affordance contract defers relation:* verbs. The four entity verbs let an app create entities but never LINK them — a severe gap for a graph tool. (PATCH update CAN carry a relations wrapper, covering some cases, but not standalone edge create/delete.) (2) ''trace'' has NO backing REST endpoint in the SPA api layer (trace exists only via MCP) — it''s aspirational. (3) ''rename'' has no dedicated REST endpoint/frontend function either (no renameEntity in entities.ts) — verify how rename is actually wired before exposing it. (4) Missing endpoints an app realistically needs: _schema + _config (MANDATORY — entities.ts:35-44 can''t even build an entity URL without the type→plural map), _templates/{type} (scaffolding create forms), _documents/{name}/{id} (renders Lua templates to HTML — arguably the single most relevant endpoint for ''apps''), dry_run create (preview validation/affordances), _position (pagination). FIX: redefine the bridge surface against the ACTUAL endpoint set: include relations ops and schema/config at minimum; drop or correct trace and rename; decide explicitly which read endpoints (templates/documents/views/position/dry_run) are in v1. Keep it a closed allow-list, but make it the right list.'
severity: significant
resolution: 'Resolved by redefining the bridge surface to full graph read+write against the ACTUAL endpoint set. v1 bridge methods: reads = schema, config, list, get, search, analyze, templates, documents, position; writes = create, update, delete, plus relations (createRelation/updateRelationProperties/deleteRelation on the /{plural}/{id}/relations/{relType}[/{targetId}] sub-resource); plus action (POST /_action/{id}). Dropped ''trace'' (no backing REST endpoint — MCP-only). ''rename'' to be verified during impl (no renameEntity in entities.ts; likely a PATCH-of-id) and included only if a real endpoint exists, else deferred. schema+config are mandatory (URL building needs the type→plural map). Still a CLOSED allow-list (each method maps to one concrete api-client call, no path passthrough) — just the correct list. Plan updated.'
status: addressed
