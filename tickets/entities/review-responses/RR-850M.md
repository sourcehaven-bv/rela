id: RR-850M
type: review-response
title: 'S1: --bind 0.0.0.0 was silently broken — operators locked out of their own server'
finding: 'newSecurity built the Host allowlist by adding only the bind address itself when the bind was non-loopback. For `--bind 0.0.0.0` this meant `0.0.0.0:8080` was the only allowed Host header. No browser ever sends Host: 0.0.0.0:8080; they send the LAN IP / mDNS name / hostname the user typed. So the moment an operator opted into LAN access, every request was 403''d. The docs/security.md snippet recommending `--bind 0.0.0.0` was non-functional. The test TestNewSecurity_NonLoopbackBindOnlyAcceptsItself codified this broken behavior as correct.'
severity: significant
resolution: 'Added isUnspecified helper. When the bind address is 0.0.0.0 / :: / empty, allowedHosts is left nil and requireLocalHost bypasses the check entirely (the Origin allowlist remains the only CSRF gate in that mode). docs/security.md updated to document this and to require operators to pass --allowed-origin for every hostname clients use to reach the server. Added TestNewSecurity_UnspecifiedBindAcceptsAnyHost covering 0.0.0.0:8080, :8080, [::]:8080. Fixed splitBindAddress to return 0.0.0.0 (not 127.0.0.1) for empty-host bind, preserving operator intent.'
status: addressed
