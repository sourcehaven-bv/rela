title: CSP response header is stripped when HTML is injected via iframe.srcdoc
finding: 'The plan serves app HTML from GET /api/v1/_apps/{id} with a hardened CSP HTTP header AND fetches that HTML to inject into iframe.srcdoc. These are contradictory: an HTTP Content-Security-Policy response header is NOT applied to content injected via iframe.srcdoc — when the host does axios.get() and assigns response.data to srcdoc, the browser only sees the response BODY; the CSP header is consumed by the fetch layer and discarded. A srcdoc document''s CSP can only come from (a) a <meta http-equiv="Content-Security-Policy"> inside the HTML string, or (b) inheritance/intersection from the parent document CSP + the iframe sandbox attribute. The existing codebase precedent (handlers_theme.go:60) is header-based and self-described as direct-navigation-only (comment at :57-59) — it does NOT carry into srcdoc, and there is NO <meta>-CSP injection pattern anywhere in the repo (grep http-equiv is empty). As written, the security model has a genuine hole: the in-iframe app would run with no effective CSP. FIX: the server must inject a <meta http-equiv=CSP> into the HTML before returning it (for the srcdoc path) AND/OR the design must rely explicitly on the iframe sandbox attribute (allow-scripts WITHOUT allow-same-origin) + a parent-page CSP, and the plan must stop claiming the response header covers the srcdoc case. Confirmed: handlers_theme.go:56-61, no http-equiv usage in codebase.'
severity: critical
resolution: 'Resolved in the plan. The server-side _apps/{id} handler will INJECT a <meta http-equiv="Content-Security-Policy"> element into the app HTML before returning it, so the CSP travels inside the body and is enforced inside the srcdoc document (where the HTTP header is stripped). Defense-in-depth retained: (a) the iframe sandbox attribute = allow-scripts WITHOUT allow-same-origin (origin-null, no cookies/localStorage/parent-DOM, no own egress); (b) the HTTP CSP header is ALSO set on the response for the (unreachable-but-harmless) direct-navigation case + as belt-and-braces; (c) per-app csp_origins are merged into BOTH the meta CSP and the header. The meta-CSP injection is net-new (no http-equiv precedent in repo) and is now an explicit implementation step with a handler test asserting the meta tag is present and well-formed in the served body.'
status: addressed
