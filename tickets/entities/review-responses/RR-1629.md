id: RR-1629
type: review-response
title: e2e tests rely on Playwright eval latency exceeding 600ms open-delay
finding: 'e2e/tests/markdown-editor-backtick-autocomplete.spec.ts uses two-step typing: `typeIntoEditor(''see `'')` then `await form.waitForBacktickPopup()` (1500ms timeout) then `typeIntoEditor(''FEAT-'')`. The `typeIntoEditor` helper (form.page.ts lines 311-321) uses `document.execCommand(''insertText'', false, c)` in a loop with NO explicit wait between steps and NO test-only override of OPEN_DELAY_MS. Today''s stability depends on (a) waitForBacktickPopup() not returning before 600ms have elapsed since the backtick was typed — it can return as soon as the popup is visible, which is delay+render+microtask; on a fast CI runner this could be exactly at the boundary, and (b) the implicit assumption that Playwright''s `page.evaluate` round-trip is slow enough that consecutive characters in a single `typeIntoEditor` call don''t all land within 600ms. Both assumptions are environment-dependent. Concretely: on a fast macOS dev runner, `for (const ch of ''FEAT-'') await page.evaluate(...)` could complete in <100ms total — inside the open-delay window. The ''typing `flag-name` quickly does NOT show the popup'' test happens to work because the hyphen disqualifies the buffer; reverse case (''FEAT-'') would also fit inside 100ms and race. Fix: expose `openDelayMs` via a test hook (e.g. window flag read at composable construction time) and set it to 50 in e2e. Or use `page.keyboard.type()` with delay option which adds explicit per-keystroke timing.'
severity: significant
resolution: MarkdownEditor.vue reads window.__BACKTICK_AUTOCOMPLETE_DELAY_MS__ at construction and passes it as options.openDelayMs to useBacktickAutocomplete when set. e2e adds form.useFastAutocompleteDelay() helper that injects this via page.addInitScript before navigation. All 6 e2e tests now use the fast delay (30ms) so they don't race the production 600ms grace period on fast CI runners. Production builds never set the window flag.
status: addressed
