id: RR-L289
type: review-response
title: NewForTest silently continues with nil search index on failure
finding: If search.NewIndex() fails in NewForTest, the test workspace has a nil search index and any test that hits Search() gets a confusing 'search index not available' error. Should fail hard (panic) since this is an internal test helper.
severity: nit
resolution: NewForTest now panics with a clear message if search.NewIndex() or idx.IndexBatch fails, instead of silently continuing with nil. This is acceptable for a test helper; if a test environment can't create an in-memory search index, all tests should fail fast.
status: addressed
