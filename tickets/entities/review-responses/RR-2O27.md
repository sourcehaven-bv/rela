title: TKT-VMD8 sidebar two-mode design is gratuitous — collapse to one GraphCount path
finding: 'Plan keeps the precomputed typeCounts cache for the non-ACL path and adds a separate ACL path using GraphCount. typeCounts today is `for type in Meta.EntityTypes(): Store.CountEntities(type)` (api_v1.go:2263). For postgres: SELECT count(*); for fsstore/memstore: linear walk — exactly what GraphCount(GraphQuery{EntityType:type}) does. No measurable perf delta. Drop both modes. Always go through ReadQuery(type): AllowAll → GraphCount(GraphQuery{EntityType:type}); DenyAll → 0; Query → GraphCount(*rqr.Query). Makes the security property structural rather than conditional, eliminates the ''did we remember to skip the cache when ACL is on'' footgun forever, shrinks the diff. filterCache keyed by listID stays (within-request memo, not cross-principal alias).'
severity: significant
status: open
