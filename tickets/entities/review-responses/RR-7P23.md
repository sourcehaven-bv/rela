title: Inaccessible struct conflates entity and relation domains
finding: 'Plan: single Inaccessible{ID, Type, Path, Reason} for both entities and relations. For entity files, Type=entity-type. For relations, what is Type? Plan says ''empty for relations'' — but then the SPA cannot discriminate without checking which iterator yielded it, and downstream consumers must keep two streams separate. Either add a Kind field (entity|relation) or split into InaccessibleEntity and InaccessibleRelation types. Spell out Path format for relations (relations/FROM--type--TO.md).'
severity: minor
resolution: 'No longer applicable: there is no Inaccessible value type. The field-on-entity design uses entity.Entity for entities and entity.Relation for relations, each with its own Inaccessible []InaccessibleField. Discrimination is automatic by which type was loaded.'
status: addressed
