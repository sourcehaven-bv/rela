id: RR-L7487
type: review-response
title: Validation reads outside WithTx racing against concurrent reload
finding: The new Workspace.Rename ran validateRename, GetNode, IncomingEdges, OutgoingEdges against w.state.Load() BEFORE entering WithTx. A concurrent Reload between those reads and WithTx loading its own snapshot could give the closure a stale entity payload to write under a possibly-newer metamodel. Pre-existing race in the old rename.go too, but the new code widens it because the entity payload travels across the snapshot boundary.
severity: critical
resolution: Fixed in this PR. Workspace.Rename now does ALL validation and edge collection inside the WithTx closure via the new loadAndValidateRename helper, which reads from tx.base.graph. The entity payload, the relation lists, and the metamodel passed to WriteEntity all come from the same in-tx snapshot. Pre-tx validation only happens in the dry-run path, where consistency is best-effort by design.
status: addressed
