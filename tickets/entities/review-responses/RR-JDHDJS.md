title: Sync endpoint auth + principal attribution undecided (security)
finding: 'The plan flags sync-endpoint auth as an ''open question'' but leaves it unresolved — yet this is a network write endpoint applying arbitrary entity content to the server, so auth is not optional. Verified plumbing exists: data-entry middleware stampAuditPrincipal (router.go:374-378) sets principal per-request via a pluggable resolver (router.go:237,277-304) deriving User from --principal-header / $RELA_DATAENTRY_USER, sanitized (sanitizeUser, router.go:333-364). So a sync push CAN attribute to the original author IF the server trusts the header — but ''trusts the header'' is exactly the security decision: an unauthenticated --principal-header is forgeable attribution (audit-log spoofing). Decisions the plan must make before implementation: (1) transport auth mechanism (bearer token / mTLS / reuse existing data-entry auth) — NOT just a trusted header; (2) Tool attribution — there is no ToolSync constant (principal.go:40-46); sync writes would currently mis-attribute as Tool=data-entry. Add a ToolSync constant. (3) principal is per-REQUEST (manager reads ctx once per call), so a batch from multiple authors can only carry one principal — decide: one-request-per-author, or attribute the whole push to the syncing user. Allowlist-validate id/kind path params (reject path-traversal-shaped ids) before they reach the store.'
severity: significant
resolution: 'Resolved by deployment model: prod puts rela-server behind an OAuth proxy (oauth2-proxy / Vouch / traefik forward-auth), which is the documented intended posture (docs/security.md). The proxy authenticates and sets X-Forwarded-User; the app trusts it because only the proxy can reach the server. Sync endpoints (under /api/sync/) inherit this gate. Server stays auth-less by design; no credential gate built here. Captured in TKT-PV0R3V (server: ToolSync constant, /api/sync/ same-origin exemption, trust-model docs) and TKT-T4H4YK (client: must traverse proxy with a bearer token via RELA_SYNC_TOKEN/--token; --remote URL; never log token).'
status: addressed
