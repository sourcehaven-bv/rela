type: review-response
title: Editor unmount during open modal leaks the captured editor reference
finding: 'Plan describes capturing the EasyMDE `editor` instance and using `editor.codemirror.replaceSelection(...)` in the modal''s `@select` handler. If the form is unmounted while the picker is open (route change, user navigates away mid-pick) the captured `editor` reference points at a torn-down CodeMirror, and `replaceSelection` either no-ops or throws on a stale DOM node. The plan does not specify: (a) whether the picker must close when its parent MarkdownEditor unmounts, (b) what happens if `@select` fires after MarkdownEditor''s `onBeforeUnmount` has set `editor = null`. Mitigation: gate the insertion on a non-null `editor` reference at the call site, and have the MarkdownEditor close the picker in `onBeforeUnmount`. Add a unit test for `insertEntityRef(null, id)` (no-op).'
severity: significant
resolution: 'Plan §Approach §2 + §3 + AC 8: null-editor guard in helper; MarkdownEditor sets pickerOpen=false in onBeforeUnmount; @select handler guards on editor != null. Helper unit test ''null editor → no-op'' added.'
status: addressed
