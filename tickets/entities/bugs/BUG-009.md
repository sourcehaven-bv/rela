---
id: BUG-009
type: bug
title: Entity types API returns empty plural labels
description: The MCP list_entity_types API was returning empty plural labels because the Plural field is unexported in Go. This caused mobile apps to show empty labels in entity type lists.
priority: high
effort: xs
why1: PluralLabel field was accessed as 'Plural' in the response
why2: Go JSON marshaling requires exported fields
why3: The struct had both Plural (unexported) and PluralLabel (exported)
why4: Two parallel field names (Plural and PluralLabel) existed without a single source of truth
why5: API response structs were not validated against an external contract test, so missing fields silently serialized as empty
prevention: Use the exported PluralLabel field consistently in API responses
status: done
---
